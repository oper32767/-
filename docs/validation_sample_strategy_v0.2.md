# 驗證樣本策略 v0.2｜靜態樣本庫＋動態樣本庫

日期：2026-06-17

## 1. 本版修正原因

使用者指出：前版把公開驗證樣本重點偏到地震、天災，這會讓動態運算層樣本過窄，也漏掉靜態運算層樣本。

本版修正為兩條樣本庫：

```text
靜態樣本庫 = 八字命例 + 紫微命例 + 生平／事件驗證資料。
動態樣本庫 = 六壬實占案例 + 奇門實戰案例 + 問事／起課／起局時間 + 已知結果。
```

---

## 2. 靜態樣本庫定位

靜態運算層驗證對象：

```text
八字完整運算 + 紫微斗數完整運算 + 八字×紫微交叉驗證。
```

因此靜態樣本不是找地震，也不是找一般事件，而是找「命例」。

### 2.1 靜態樣本來源優先序

| 優先 | 來源類型 | 中文說明 | 可用性 |
|---|---|---|---|
| S1 | 公開教學案例 | 老師在課程、文章、影片中引用某人八字／紫微命盤，並用生平事蹟驗證 | 高 |
| S2 | 古籍命例 | 《滴天髓》《三命通會》《子平真詮》等命例，或後人註解命例 | 高，但需校對年代與時辰 |
| S3 | 社群自貼命例 | 使用者自己貼出生年月日時、八字、紫微命盤並問事，有後續回饋 | 中 |
| S4 | 名人命例 | 名人出生資料＋公開生平 | 中，但出生時間可靠性需分級 |
| S5 | 不完整命例 | 只有年月日、沒有時辰，或只有盤截圖 | 低，只作候選或局部測試 |

### 2.2 靜態樣本必備欄位

| 欄位 | 中文說明 | 缺失處理 |
|---|---|---|
| sample_id | 樣本編號 | 必填 |
| source_type | 來源類型 | 必填 |
| source_url | 來源網址 | 必填 |
| birth_datetime | 出生年月日時 | 缺時辰則 WARN（警告）或 BLOCK（阻擋） |
| birth_place | 出生地 | 缺則 WARN（警告） |
| gender | 性別 | 缺則 WARN（警告） |
| bazi_chart | 八字盤 | 可由出生資料重排 |
| ziwei_chart | 紫微盤 | 可由出生資料重排 |
| known_life_events | 已知生平事件 | 必填，否則不能驗證 |
| event_dates | 事件日期 | 越精準越好 |
| expected_validation_axis | 驗證軸：財、官、婚姻、疾病、遷移等 | 必填 |
| reliability_grade | 可靠度分級 | 必填 |
| notes | 備註 | 選填 |

### 2.3 靜態樣本 Gate（檢核閘門）

| Gate（檢核閘門） | 判定 |
|---|---|
| 沒有出生資料 | BLOCK（阻擋） |
| 沒有生平或事件回饋 | BLOCK（阻擋） |
| 只有命盤但無來源 | BLOCK（阻擋） |
| 只有年月日沒有時辰 | WARN（警告），只能局部測試 |
| 名人出生時間來源不明 | WARN（警告） |
| 社群貼文無後續回饋 | REVIEW（待審），不入正式樣本 |
| 教學案例引用未附原始資料 | REVIEW（待審） |

---

## 3. 動態樣本庫定位

動態運算層驗證對象：

```text
六壬完整起課／解課 + 奇門完整起局／解局 + 六壬×奇門事件交叉驗證。
```

動態樣本不限定天災。只要有足夠時間點與結果，就可以候選。

### 3.1 動態樣本來源優先序

| 優先 | 來源類型 | 中文說明 | 可用性 |
|---|---|---|---|
| D1 | 實占教學案例 | 影片、課程、文章中有問事時間、起課時間、起局時間、結果 | 高 |
| D2 | 社群問事案例 | 網友問事，有明確發文時間／問事時間與後續結果 | 中高 |
| D3 | 公開事件案例 | 體育賽事、公告事件、交通事件、天氣事件、地震等 | 中 |
| D4 | 即時新聞事件 | 有明確發生時間與結果 | 中，但需避免事後硬套 |
| D5 | 無結果案例 | 只有起局盤、沒有結果 | 低，不入正式驗證 |

### 3.2 動態樣本必備欄位

| 欄位 | 中文說明 | 缺失處理 |
|---|---|---|
| sample_id | 樣本編號 | 必填 |
| source_type | 來源類型 | 必填 |
| source_url | 來源網址 | 必填 |
| question_text | 問題文字 | 必填 |
| question_time | 問事時間 | 必填或需可推定 |
| divination_time | 起課／起局時間 | 必填 |
| timezone | 時區 | 必填 |
| location | 起課／起局地點 | 缺則 WARN（警告） |
| known_result | 已知結果 | 必填 |
| result_time | 結果發生時間 | 越精準越好 |
| usable_for | 六壬、奇門或兩者 | 必填 |
| reliability_grade | 可靠度分級 | 必填 |
| notes | 備註 | 選填 |

### 3.3 動態樣本 Gate（檢核閘門）

| Gate（檢核閘門） | 判定 |
|---|---|
| 沒有問事時間也沒有起課／起局時間 | BLOCK（阻擋） |
| 沒有結果 | BLOCK（阻擋） |
| 時區不明 | WARN（警告） |
| 起局時間與事件發生時間混淆 | BLOCK（阻擋） |
| 只有盤圖沒有文字來源 | REVIEW（待審） |
| 天災、新聞、賽事類事件可用，但不得成為唯一類型 | WARN（警告） |

---

## 4. 收束規則

為避免樣本庫放太大，第一階段只收以下數量：

| 樣本庫 | 第一階段數量 | 來源限制 |
|---|---|---|
| 靜態樣本庫 | 10 筆 | 至少 5 筆要有生平事件驗證 |
| 動態樣本庫 | 10 筆 | 至少 5 筆要是實占／社群案例，不可全是天災 |

---

## 5. 第一階段搜尋關鍵詞

### 5.1 靜態樣本搜尋詞

```text
八字 案例 生辰 生平
八字 命例 生平 教學
八字 實例 生辰 事蹟
紫微斗數 案例 生辰 生平
紫微斗數 命例 事蹟 教學
名人 八字 生辰 事件
```

### 5.2 動態樣本搜尋詞

```text
六壬 實占 案例 起課時間 結果
六壬 問事 案例 回饋
大六壬 實戰案例 三傳 結果
奇門遁甲 實戰案例 起局時間 結果
奇門 問事 案例 回饋
奇門遁甲 實例 時間 結果
```

```text
先找案例，不先擴大工具。
先收 10+10，不做無限樣本庫。
靜態層與動態層分開收樣本，再進四術交叉。
```

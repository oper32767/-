# AI Agent（代理程式）分工草案 v0.1

日期：2026-06-17

## 1. 目的

把 AI 工具、Agent（代理程式）、GitHub（版本控制／程式碼倉庫）、NotebookLM（Google 筆記型 AI 工具）、Google Drive（雲端硬碟）與網路公開資料分工，不再全部靠單一聊天串人工補。

---

## 2. 基本分工

| Agent（代理程式） | 中文定位 | 可做 | 禁止 |
|---|---|---|---|
| Source Agent | 來源蒐集員 | 搜尋官方資料、公開事件、GitHub（程式碼倉庫）文件 | 不作術數主判 |
| Intake Agent | 收件整理員 | 把來源整理成樣本卡 | 不改規則 |
| Gate Agent | 檢核員 | 檢查時間、地點、來源、結果是否齊全 | 不硬補缺失資料 |
| Static Agent | 靜態運算員 | 跑八字×紫微靜態運算層 | 不處理臨場起課起局 |
| Dynamic Agent | 動態運算員 | 跑六壬×奇門動態運算層 | 不取代靜態層年度判斷 |
| Evidence Agent | 證據員 | 把結果回填表格與 GitHub（版本控制／程式碼倉庫） | 不宣稱封板 |
| Review Agent | 複核員 | 比對前後版本、找矛盾 | 不直接覆蓋舊版本 |

---

## 3. 最小流程

```text
公開來源搜尋
→ 樣本卡整理
→ Gate（檢核閘門）檢查
→ 六壬／奇門起課起局
→ 結果與已知事件比對
→ 回填樣本庫
→ 寫入 GitHub（版本控制／程式碼倉庫）
→ 必要時更新糾正鏈與 Skill（技能）
```

---

## 4. 權限分級

| 層級 | 權限 | 說明 |
|---|---|---|
| READ（讀取） | 可讀來源 | 低風險 |
| WRITE_DRAFT（寫草稿） | 可寫候選檔 | 不得覆蓋正式檔 |
| WRITE_PATCH（寫補丁） | 可新增補丁版 | 不得刪舊版 |
| WRITE_MAIN（寫主線） | 可改主線 | 需雲龍明確確認 |
| DELETE（刪除） | 刪檔 | 預設禁止 |

---

## 5. 工具對應

| 工具 | 中文用途 | 目前狀態 |
|---|---|---|
| GitHub Connector | GitHub 連接器／程式碼倉庫寫入 | 已連上，已可寫入 |
| Google Drive Connector | 雲端硬碟連接器 | 可建資料夾，但上傳受容量限制 |
| NotebookLM | Google 筆記型 AI 工具 | 需手動匯入第 61 串 |
| Web Search | 網路搜尋 | 用於查官方資料與公開事件 |
| Spreadsheet | 試算表 | 用於核心表、糾正鏈、樣本庫 |
| Apps Script | Google 試算表腳本 | 草案中，未乾跑封板 |

---

## 6. 目前優先任務

1. 用 GitHub（版本控制／程式碼倉庫）保存主線骨架。
2. 建立公開事件樣本庫。
3. 把第 61 串手動匯入 NotebookLM（Google 筆記型 AI 工具）。
4. 先用 5～10 筆公開事件跑動態層測試。
5. 每次失誤回填 Gate（檢核閘門）與糾正鏈。

---

## 7. 禁止事項

1. 不得讓 Agent（代理程式）直接覆蓋正式規則。
2. 不得用單一術數覆蓋交叉驗證。
3. 不得把候選樣本寫成正式命中率。
4. 不得在缺時間、地點、結果時起課起局。
5. 不得把 GitHub（版本控制／程式碼倉庫）當垃圾桶；每個檔案都要有用途。

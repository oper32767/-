## 1. 本倉庫定位

本倉庫不是術數主判答案庫，也不是封板規則庫。

它先做三件事：

1. 記錄目前主線現況。
2. 固定靜態運算層與動態運算層的正確分工。
3. 建立 AI 工具與公開驗證資料的後續施工骨架。

## 2. 目前核心裁定

```text
靜態運算層 = 八字完整運算 + 紫微斗數完整運算 + 八字×紫微交叉驗證。
動態運算層 = 六壬完整起課／解課 + 奇門完整起局／解局 + 六壬×奇門事件交叉驗證。
```

不得把「八字」或「紫微」單獨叫完整靜態層；兩者完整算完並交叉驗證後，才叫靜態運算層。

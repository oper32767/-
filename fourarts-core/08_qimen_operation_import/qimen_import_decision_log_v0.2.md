# 奇門外部搬回與內部主鏈接入裁定紀錄 v0.3

## 0. 本輪範圍

本輪已從「外部來源候選整理」推進到：

1. 六十一串內部奇門主鏈機器映射；
2. 多引擎統一輸出格式；
3. 第一分歧差分工具；
4. 固定回放與邊界案例；
5. 工具單元測試；
6. GitHub Actions（GitHub 自動化工作流程）。

本輪仍不修改《四術算法依據庫 v1.0》，不新增正式 `rule_id`（規則編號），不自動合併。

## 2. v0.3 新增裁定

### D-009｜六十一串主鏈可以轉成機器設定

**裁定：採用。**

已接入：

- 時家奇門；
- 轉盤奇門；
- 置閏主鏈；
- 下北上南；
- 從零重建；
- 中五寄坤，寄星與寄干同步；
- 異常摘要先於解盤；
- 日干為求測人、時干為問事；
- 用神落宮整包；
- 排盤、解盤、運籌三主鏈分離。

未鎖死：

- 晚子時唯一換日政策；
- 暗干四法的最終單一裁決。

處理：列入 `Unknown List`（未知清單），未聲明時停機。

### D-010｜建立多引擎標準輸出

**裁定：採用。**

理由：不同引擎輸出格式不同。若不先標準化，就無法確認差異來自曆法、定局、寄宮、暗干或盤面結構。

新增：`qimen_engine_output_schema_v0.1.json`。

### D-011｜建立第一分歧差分工具

**裁定：採用。**

新增：`tools/qimen_diff_harness.py`。

`harness`（測試支架）只做逐層比較，不判定正誤。

第一分歧順序：

1. 時區、真太陽時、晚子時與換日；
2. 節氣、四柱、旬首；
3. 陰陽遁、局數、三元；
4. 地盤；
5. 值符、值使；
6. 天盤、九星、八門、八神；
7. 暗干、空亡、馬星；
8. 異常。

### D-012｜外部基準盤不得偷補時區

**裁定：採用。**

外部測試向量若未明示時區，只能在明確標記「回放假設」後測試，不得直接進正式判定。

### D-013｜差分工具必須有自己的測試

**裁定：採用。**

新增單元測試：

- 相同輸出應回報 `PASS`（通過）；
- 局數不同時，第一分歧應為 `ju.number`（局數欄位）。

### D-014｜加入 GitHub Actions（GitHub 自動化工作流程）

**裁定：採用。**

每次候選包變更，自動執行：

- 結構驗證；
- 內部主鏈口徑驗證；
- 差分工具單元測試。

此流程只保證施工檔案沒有結構性退化，不代表術數正確。

## 3. 隔離裁定維持不變

### 外部單宮加權分數

固定權重與空亡後減半等設計，可作測試，但來源與權重未充分證明。

裁定：

> `SCORING_EXPERIMENT_ONLY`（只作評分實驗）。

### 外部用神與運籌敘事

缺逐條來源、混入現代敘事、操控或破壞性描述者，不搬規則內容。

### 自訂均分法

作者明示為原創，只能列實驗方法。

裁定：

> `CUSTOM_METHOD_HOLD`（自訂方法暫停）。

### 近期仍在修錯的置閏引擎

可作差分訊號，不作正式基準。

## 4. 目前可用程度

| 層級 | 狀態 | 能做什麼 |
|---|---|---|
| 來源治理 | 可用候選 | 記錄授權、提交碼、源流與獨立證據 |
| 主鏈映射 | 已完成第一版 | 將六十一串主鏈轉為機器設定 |
| 差分輸入 | 已完成 | 接收標準化引擎輸出 |
| 差分比較 | 可執行 | 找出第一個分歧與全部差異 |
| 內部自動排盤 | 尚未完成 | 仍不能只輸入時間便自動排完整盤 |
| 正誤裁決 | 人工 Gate（閘門） | 需回六十一串與原典裁決 |
| 高干預運籌 | `HOLD`（暫停） | 不准直接用於真人案例 |

## 5. 三項主要盲點

1. 外部基準盤可能共用同一 App（應用程式）或同一算法，未必真正獨立。
2. 開源程式的測試通過，只證明符合自己的預期，不證明術數規則正確。
3. 古籍轉錄、OCR（光學文字辨識）與現代資料表都可能有二次加工錯誤。

# 奇門外部搜尋與搬回裁定紀錄 v0.2

## 0. 本輪範圍

本輪搜尋三類：

1. 排盤引擎與曆法；
2. 解盤規則、用神及測試；
3. 運籌模組與資料結構。

本輪只搬回：

- 可重跑的測試向量；
- 方法與曆法契約；
- 差分校驗框架；
- 解盤證據契約；
- 公版古籍來源地圖；
- 來源、版本與授權登記。

## 2. 不採用或隔離

### `qfdk/qimen` 的單宮加權分數

它將門、神、星宮、門宮、十干克應等以固定權重加總，並將空亡後的總分減半。這種設計可測試，但來源與權重未充分證明，不能搬成正式解盤算法。

裁定：

> `SCORING_EXPERIMENT_ONLY`（只作評分實驗）。

### `perfhelf/bigfishmarquis-qimen` 的用神與運籌敘事

資料量大，但部分條目：

- 缺逐條原典定位；
- 混入現代敘事；
- 含操控、散布虛假資訊或破壞性描述；
- 自述驗證未見測試證據。

裁定：

> 只保留條件資料結構，不搬規則內容。

### `Taogram/taobi` 的均分法

作者明示為原創。可作實驗方法，但不能當傳統共識。

裁定：

> `CUSTOM_METHOD_HOLD`（自訂方法暫停）。

### `redrockhorse/qimenpaipan`

近期提交仍在修正置閏與三元錯誤。

裁定：

> 可作差分訊號，不作正式基準。

## 3. 三項盲點

1. 外部基準盤可能共用同一 App（應用程式）或同一算法，未必真正獨立。
2. 開源程式的測試通過只證明符合自己的預期，不證明術數規則正確。
3. 古籍轉錄與現代資料表都可能有抄寫、OCR（光學文字辨識）或二次加工錯誤。

## 4. 驗證方法

1. 用標準化曆法快照鎖定輸入；
2. 至少三個引擎排同一盤；
3. 分層比對到第一個差異點；
4. 回到六十一串或公版古籍查規則；
5. 建立人工裁定；
6. 把裁定加入回歸測試。

## 5. 量化風險

| 項目 | 風險估計 |
|---|---:|
| 把不同口徑誤判為程式錯誤 | 40%～60% |
| 把同源程式誤算為獨立驗證 | 30%～50% |
| 外部解盤資料含無來源敘事 | 60%～80% |
| MIT 基準向量可直接沿用為測試候選 | 80%～90% |
| 外部程式可直接升正式算法 | 低於20% |

## 6. 備援

- 所有新增內容維持 Draft PR（草稿合併請求）；
- 正式規則庫不動；
- 外部向量若驗證失敗，只刪除候選，不影響內部主鏈；
- 每個來源保留提交碼與授權，必要時可完整回退。

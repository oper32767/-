# 奇門排盤差分校驗框架 v0.1

> 狀態：`ENGINEERING_CANDIDATE`（工程候選）  
> 目的：把「兩個軟體結果不同」拆成可定位的差異，不用整盤互相覆蓋。

## 1. 核心原則

1. 先統一輸入，再比較輸出。
2. 先比曆法層，再比定局層，最後才比排盤層。
3. 任一上游層不同，下游差異不得直接判成引擎錯誤。
4. 任何引擎都不是天然真相源；正式主判仍以六十一串封板規格與人工裁決為準。
5. 同源引擎不能當成兩個獨立證據。

## 2. 最低比較單位

每次差分必須保存：

- 標準化 `QimenCalendarSnapshot`（奇門曆法快照）；
- 原始輸入時間、時區、地點、真太陽時政策；
- 晚子時換日政策；
- 節氣交界時間與距離；
- 排盤法、盤式、寄宮、暗干算法；
- 各引擎版本、提交碼及輸出雜湊。

缺一項即輸出 `HOLD_INPUT_NOT_NORMALIZED`（輸入未正規化，暫停）。

## 3. 分層比較順序

### Stage 0（階段零）｜時間層

比較：民用時間、時區、經緯度、真太陽時、晚子時同日／次日、日界線。不同則停止，不進下一層。

### Stage 1（階段一）｜曆法層

比較：

- 儒略日；
- 當前節氣；
- 節氣交接時刻；
- 年月日時四柱；
- 旬首；
- 符頭與三元。

允許為每個欄位設定誤差，例如節氣時間可記秒差，但不得先強行取整。

### Stage 2（階段二）｜定局層

比較：

- 陰遁／陽遁；
- 局數；
- 上中下元；
- 拆補／置閏／茅山等方法；
- 超神、接氣、置閏狀態。

方法不同時，標記 `METHOD_DIVERGENCE`（方法分歧），不是程式錯誤。

### Stage 3（階段三）｜地盤與符使

比較：三奇六儀地盤、旬首遁干、值符原宮、值符星、值使門、值符與值使落宮。

### Stage 4（階段四）｜天、人、神盤

比較：天盤干、九星、八門、八神、天禽寄宮及中宮處理。

### Stage 5（階段五）｜補充盤面

比較：暗干、空亡、馬星、擊刑、入墓、門迫、伏吟、反吟及其他流派擴充異常。

異常分類必須聲明：

- `YINPAN_FOUR_HARMS`（陰盤四害）；
- `XUNSHUANG_SIX_HARMS`（荀爽六害）；
- 或其他已明示口徑。

### Stage 6（階段六）｜解盤資料

只比較機械條件，不比較文風：

- 用神選取條件；
- 本人與對方落宮；
- 主客判定；
- 宮際生剋；
- 格局命中；
- 支持、反對及硬性禁令。

## 4. 差異代碼

| 代碼 | 意義 |
|---|---|
| `TIME_POLICY_DIVERGENCE` | 時間或換日口徑不同 |
| `SOLAR_TERM_DIVERGENCE` | 節氣來源或邊界不同 |
| `GANZHI_DIVERGENCE` | 四柱或旬首不同 |
| `METHOD_DIVERGENCE` | 拆補、置閏、茅山等方法不同 |
| `LODGING_DIVERGENCE` | 天禽寄宮不同 |
| `HIDDEN_STEM_DIVERGENCE` | 暗干算法不同 |
| `ENGINE_DEFECT_SUSPECTED` | 同口徑下機械結果仍不同 |
| `SOURCE_RULE_CONFLICT` | 規則來源互相衝突 |
| `UNTRACEABLE_OUTPUT` | 無法追查輸出來源 |

## 5. 邊界測試矩陣

最低測試：

1. 每個節氣交界前後2分鐘、1小時、1天；
2. 冬至、夏至；
3. 22:59、23:00、23:59、00:00、00:59；
4. UTC+8（世界協調時間加八小時）與其他時區同一絕對時刻；
5. 同一地點民用時間與真太陽時；
6. 旬首切換；
7. 上中下元切換；
8. 天禽為值符或落中宮；
9. 伏吟、反吟；
10. 陰陽遁一至九局全覆蓋。

## 6. 驗收門檻

排盤引擎進入正式候選前，至少：

- 24節氣邊界測試全通過；
- 18局各至少3例；
- 晚子時兩種政策皆有測試；
- 天禽寄宮各口徑有明確測試；
- 暗干、空亡、馬星及四害欄位可追查；
- 與至少兩個不同曆法來源的引擎差分；
- 所有差異都有代碼與人工裁定紀錄。

## 7. 退路

發現分歧時：

1. 保留所有原始輸出；
2. 回退到最後共同一致的層；
3. 不產生解盤與運籌；
4. 將案例送入 `Unknown_List`（未知清單）；
5. 由雲龍或冷審決定採哪一口徑。

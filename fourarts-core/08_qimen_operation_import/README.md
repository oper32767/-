# 奇門排盤・解盤・運籌外部候選搬回包 v0.2

> 狀態：`EXTERNAL_CANDIDATE`（外部候選）  
> 日期：2026-07-22  
> 適用倉庫：全術數 60+1 主線總控倉庫

## 0. 定位

本目錄只搬回**可驗證、可重跑、可審計的工程做法、測試向量與公版來源導航**，不把外部術式直接升為正式奇門規則。

本包不修改：

- 《四術算法依據庫 v1.0》；
- 六十一串既有封板；
- 奇門排盤與解盤正式主鏈；
- 任何正式 `rule_id`（規則編號）。

## 0.1 來源源流修正

經雲龍依自身學習來源確認：

- 陰盤奇門普遍所稱「四害」為：**擊刑、入墓、門迫、空亡**。
- 荀爽 YouTube（影音平台）教學另形成「六害」口徑：**四害＋白虎＋庚**。
- 六十一串內的六害與化氣陣內容，來源於雲龍對荀爽教學的自學、整理、實盤與糾錯。
- `skyfiredao/qmenpowers` 的六害／化氣陣內容與六十一串屬同一上游教學源流，不得當成獨立術數佐證。
- 尋時借運、移星換斗等內容高度可能來自陰盤奇門教學體系；確切教師與影片尚未逐項鎖定。

因此本包分開記錄：

1. `UPSTREAM_TEACHING_LINEAGE`（上游教學源流）；
2. `INDEPENDENT_ENGINEERING_IMPLEMENTATION`（獨立工程實作）；
3. `INDEPENDENT_EVIDENCE`（獨立證據）。

## 1. v0.1 已搬回內容

1. 確定性排盤／運算與 AI（人工智慧）解讀分離。
2. 奇門操作模組獨立路由，不與排盤、解盤混層。
3. 先輸出完整原始結果，再做文字解讀。
4. 使用 JSON（結構化資料）保存輸入、盤面、規則來源、候選動作與停機原因。
5. 類象不得由模型自由聯想，必須綁定來源欄位。
6. 擇時可以採候選枚舉與排序，但不得把單一簡化分數當成正式吉凶。
7. 排盤校驗未通過、題型未鎖、用神未定、主客未判時，禁止進入運籌。
8. 外部來源、授權條款、版本、來源提交碼與上游源流必須留痕。

## 2. v0.2 新增內容

### 排盤層

- 標準化曆法快照契約；
- 晚子時、真太陽時、節氣邊界及寄宮口徑必填；
- 方法適配器登記；
- 分層差分校驗框架；
- qfdk MIT（寬鬆軟體授權）外部基準盤候選向量；
- 節氣、18局、晚子時、天禽寄宮及暗干的邊界測試矩陣。

### 解盤層

- 解盤規則證據契約；
- 條件、例外、衝突優先序與來源必須機器可讀；
- 模型生成內容不得直接升 `VERIFIED_RULE`（核定規則）；
- 外部加權分數與無來源斷語隔離。

### 運籌層

- 延續 v0.1 的路由、Gate（閘門）、候選評分及風險分級；
- 公版《遁甲演義》來源地圖，先追回主客、三遁、符使及異常的原始文本；
- 同源教學禁止重複計票。

## 3. 明確不搬回內容

- 「荀爽六害總數最少即為最佳時局」的單一評分法；
- 用生日局直接處理所有事件與布陣；
- 指定物件、顏色、雕像或文字必然改變現實結果；
- 「三個月起效」等未經驗證期限；
- 把布置物件、移動方位或想像行為當成已證實因果；
- 外部版太乙穿壬遁、尋時借運、移星換斗直接升正式規則；
- 未附逐條來源的用神、格局與運籌敘事；
- 以程式測試通過推導術式已被現實驗證。

## 5. 使用限制

- 本目錄全部內容均為候選，不得標為 `VERIFIED_RULE`（核定規則）。
- 外部工程做法可以試作；外部術數規則必須回到六十一串或公版原典逐條對帳。
- 高干預型操作一律先掛 `HOLD`（暫停），不得直接對真人案例執行。
- 程式化輸出必須保留盤面版本、規則版本、來源、上游源流及雜湊。
- 同一上游教材的影片、六十一串與 GitHub（程式碼託管平台）實作不得重複計票。
- 外部基準盤時間若未明示時區，不得直接作正式判定。

## 6. 驗證指令

在本目錄執行：

```bash
python3 tools/validate_qimen_import_package.py
```

本工具只驗證候選包的欄位、來源、源流與安全限制，不裁決術數正誤。
